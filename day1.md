*1장 컴파일과 컴파일러 사용법(2021.11.22)*

핵심포인트       
~~~
1. 프로그램은 일의 순서를 나열환 것이다.
2. C언어는 유닉스에 사용하기 위해 만들었다.
3. 컴파일이란 소스 코드르 컴파일러를 통해 컴퓨터가 이해하는 기계어로 만드는 과정이다.
4. 비주얼 스튜디오는 다양한 컴퍼일러 중 하나이다.
5. 컴파일하고 실행하기는 각각(c+sB->컴파일)과(c+f5->실행) 단축기로 할 수 있다.    
~~~

컴파일과정 3단계    
~~~
소스파일 ---(전처리)---> 전처리된 소스파일 ---(컴파일)---> 개제파일 ---(링크)---> 실행 파일
                                                                   startup code
1단계 전처리 단계 전처리 과정은 전처리 지시자에 따라 소스 파일을 가공하는 과정입니다.
2단계 컴파일 컴파일은 전처리가 끝난 파일을 컴파일 하면 개체파일이 됩니다. 
     - 개체파일은  CPU가 해석할 수 있는 명령어들로 이루어진 기계어 파일이지만 바로 실행하는 것은 불가능하기 때문에 운영체제가 인식할 수 있는 형태로 바꿔줘야 합니다.
3단계 링크 개체 파일에 startup code를 결합하는 과정을 수행하는데 이 과정을 링크라고 합니다. 
     - startup code는 프로그램을 실행하기 전에 필요한 준비작업을 수행하며 main 함수를 호출하여 우리가 작성한 프로그램의 코드가 실행되도록 합니다. 
        따라서 프로그램에는 항상 main 함수가 있어야 합니다.
~~~

2-1 장 C 프로그램의 구조와 데이터 출력 방법

핵심포인트    
~~~
1. C 프로그램은 main 함수로 시작한다.
2. //는 한 줄 주석문이고 /**/는 여러 줄을 한꺼번에 주석 처리하는 주석문이다.
3. printf 함수는 데이터를 화면에 출력할 때 사용한다.
4. 제어 문자를 문자열 안에 포함시키면 그 기능에 따라 출력 형태가 바뀐다.
5. printf 함수로 숫자를 출력할 때는 정수 %d, 실수는%lf 변환문자를 사용한다.
~~~

표로 정리       
~~~
문자열    printf("HI");            HI 출력
제어문자  printf("HI\n");          HI 출력 후 줄 바꿈
정수      printf("%d", 10);       정수 10 출력(변환문자 %d)
실수      printf("%lf", 3.5);     실수 3.500000 출력 (변환문자 %lf)
수식      printf("%d", 10 + 20);  10과 20을 더한 결과인 30출력
~~~

제어문자 종류    
~~~
\n 줄을 바꾼다. 
\t 출력 위치를 다음 탭 위치로 
\r 출력 위를 줄의 맨 앞으로 옮긴다.
\b 출력 위치를 한 칸 왼쪽으로 옮긴다.
\a 벨소리를 낸다.
~~~

2-2장 상수와 데이터 표현 방법


핵심포인트    
~~~
1. 상수는 그 값을 바꿀 수 없으며, 10은 정수 상수, 10.0은 실수 상수, 'a'는 문자 상수 "a"는 문자열 상수이다.
2. 정수 상수의 양수는 4바이트 크기의 2진수로, 음수는 2의 보수로 컴파일 된다.
3. 실수는 IEEE 754 표준의 double 형식에 따라 번역되며, 첫 비트는 부호 비트, 이후 11개 비트는 지수부, 나머지 52 비트는 소수부를 나타낸다.
~~~

표로정리(종류,표현 방법,사용 예)   
~~~
정수       0~9, +, -, 기호 사용                 10, -5, +20, 0
실수       0~9, +, -, .(소수점) 기호 사용       3.4, -1.7, .5, 10.0
문자       문자를 작은 따옴표로 묶음            'A', 'b', '0', '*', 
문자열     하나이상의 문자를 큰따옴포로 묶음     "A", "banana"
~~~

진법별 수의 표현 방법    
~~~
10진수   8진수   16진수   2진수
0        00      0x0     0
1        01      0x1     1
8        010     0x8     1000
10       012     0xa     1010
15       017     0xf     1111
~~~
